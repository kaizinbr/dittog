Sobre o DittoG

O DittoG é um site criado em Node.js para fins unicamente acadêmicos. O objetivo da criação do site foi mostrar o que foi aprendido durante o curso Técnico em Informática no IFPB-JP e servir de portfólio, já que pretendo seguir a área de desenvolvimento web no mercado de trabalho. O site foi criado utilizando apenas HTML, CSS, JavaScript, algumas bibliotecas externas e a RAWG API.

Além de HTML, CSS e Javascript, o corpo do website foi montado utilizando elementos gráficos criados no framework Figma e elementos da API SwiperJs, já que o foco foi desenvolver o design do Frontend. Para ícones, foram utilizadas as bibliotecas Google Fonts e Boxicons, que são gratuitas, manipuláveis e de fácil utilização.

Criado sobre Node.js utilizando o framework Express, a principal fonte do site é a RAWG API, uma API de jogos gratuita que contém informações sobre mais de 500.000 jogos, desenvolvedores, empresas e etc.

Para desenvolver a User Interface do website, busquei manter uma paleta de cores limitada, sem muitos constrantes ou cores chamativas. Por isso, um traço menos marcante que decidi adicionar foi o Blur, que, a grosso modo, é dar transparência a elementos que sobrepõem outros e dar a eles um aspecto "borrado" ou "embaçado". Utilizando apenas CSS, foi possível adicionar isso em componentes como a barra superior de navegação, o footer (rodapé) e no plano de fundo das páginas dos jogos (o que é feito dinâmicamente com Javascript).

Embora o intuito inicial fosse focar nos formatos apenas para desktops (computadores tradicionais), decidi ajustar a responsividade para dispositivos mobile menores como celulares e tablets. Usando media querys no CSS, pude ajustar os formatos para os tamanhos de telas padrões como os com 320px, 425px e 768px de largura, e outros formatos e proporções como 16/9 e 18/9.

As cores definidas foram Eerie Black, Cultured e Steel Teal, e suas respectivas variações. Com quase todos os elementos definidos em tons de Eerie Black, o site se torna agradável aos olhos e permite uma leitura clara dos conteúdos. Também pensando na leitura, a fonte principal utilizada foi a fonte Poppins, disponibilizada gratuitamente no Google Fonts.

Quanto à dinamicidade, quase todas as páginas têm algo que é adicionado ou alterado pelo Javascript, que, através de requisições HTTP, busca as informações na API RAWG e as adiciona ao HTML. Por ser um sistema simples e que depende de um servidor externo, algumas vezes pode ocorrer demora na adaptação do conteúdo, mas não é algo que não possa ser otimizado em algum momento. Pelo foco deste projeto ser a elaboração do Frontend, o maior foco ficou na estilização dos cards e elementos.

Criado para fins unicamente acadêmicos.
